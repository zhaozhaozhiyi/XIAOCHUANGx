# 问答策略说明

当前 Run 的 `binding.mode` 表示问答策略。主产品路径使用 `auto`，由 `skill-qa` 根据问题复杂度、上下文和用户表达自动选择轻量回答或深度研究；不要要求用户手动选择快速/深度。

| mode | 说明 |
|------|------|
| `auto` | 默认策略：简单问题快速直答，复杂任务自动进入深度研究；用户明确要求“简短/深入”时尊重其偏好 |
| `fast` | 优先响应速度，先结论后要点，轻量分析 |
| `deep` | 深度分析：由 Agent 按问题复杂度在「分步推理 + 结论」与「完整研究流程（检索→分析→成文）」之间自行选择；复杂任务可落工作区报告 |

> `fast` / `deep` 仅作为旧会话、调试入口和高级覆盖保留；历史 API 若传入 `research`，运行时等价 `deep`。
