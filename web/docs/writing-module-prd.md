## 1. 目标与范围

### 1.1 目标

根据五类报告模板与参数，经 Agent 生成含数据图表的研究文稿；支持多步骤（大纲确认）与快速两种流程；产出落 `projectId` 工作区，「我的文稿」为 Nest 索引 + 工作区正文权威（§6.0.4）。

### 1.2 版本

| 阶段 | 范围 |
|------|------|
| **MVP（v3.0）** | 导航占位，**不验收** |
| **V1.1** | 五类模板、多步骤 + 快速、我的文稿、Word/PDF 导出、工作区文件验收 |

---

## 2. 用户故事

| ID | 故事 | 优先级 |
|----|------|--------|
| WR-01 | 选择报告类型并设置参数，生成符合模板的研究报告 | P0 |
| WR-02 | 多步骤模式下确认大纲后再撰写，控制结构 | P0 |
| WR-03 | 快速模式用最简参数尽快获得初稿 | P1 |
| WR-04 | 报告中嵌入与对话一致的数据图表 | P0 |
| WR-05 | 在「我的文稿」查看进度、失败重试、继续编辑 | P0 |

## 3. 信息架构

| 二级菜单 | 路由（原型） | 说明 |
|----------|--------------|------|
| 政策解读 / 专题 / 行业 / 宏观 / 行业数据 | `/writing/{templateId}` | 五类模板入口 |
| 我的文稿 | `/writing/mine` | 任务列表与继续编辑 |

`templateId` → Skill：`skill-wr-{templateId}`（见 `WRITING_TEMPLATE_SKILL`）。

---

## 4. 端到端流程

```mermaid
flowchart LR
  P[参数 / 模板] --> D[研究方向 optional]
  D --> O[大纲确认]
  O --> W[Agent 撰写]
  W --> F[工作区 MD/DOCX]
  F --> E[导出 Word/PDF]
```

| 模式 | 步骤 |
|------|------|
| **多步骤** | 参数 → 研究方向 → 大纲确认 → 撰写 |
| **快速** | 最简参数 → 大纲（可跳过）→ 撰写 |

## 5. 功能需求

### 5.1 模板与 Skill（WR-001）

| `templateId` | 标签 | 流程 Skill |
|--------------|------|------------|
| `policy` | 政策解读 | `skill-wr-policy` |
| `topic` | 专题研究 | `skill-wr-topic` |
| `industry` | 行业研究 | `skill-wr-industry` |
| `macro` | 宏观数据解读 | `skill-wr-macro` |
| `sector-data` | 行业数据解读 | `skill-wr-sector-data` |

横切注入：`skill-platform-research-norms`。

### 5.2 工作区与项目（WR-002）

| 项 | 要求 |
|----|------|
| 创建 | 进入撰写 → 会话 + `projectId`；未选课题 → `XIAOCHUANG/写作/{YYYY-MM-DD}/{标题}/` |
| 产出 | `outline.md`、章节文稿、导出物写入 `projectId` 根 |
| 我的文稿 | Nest 索引（标题、状态、时间）；详情 **优先**读工作区 canonical 文件（§6.0.4） |
| 分支 | **无**对话式分支；重新生成 = 同任务内操作或新建任务 |

### 5.3 导出与图表（WR-003）

- 正文富文本/Markdown + 至少 1 处图表（F-QA-003）
- 导出 Word、PDF；可选 Excel 源数据

### 5.4 我的文稿（WR-004）

状态：草稿 / 生成中 / 已完成 / 失败；支持删除、复制为新草稿、单章重试。

| 文稿任务索引 | Nest |

## 7. 验收清单（V1.1）

- [ ] 五类模板均可走完多步骤或快速流程并导出
- [ ] 大纲修改后正文结构与大纲一致
- [ ] `projectId` 根至少 1 个可预览文件（F-RT-004）
- [ ] 失败任务在「我的文稿」可见且可重试
- [ ] 生成中离开页面，历史列表可发现异步结果
