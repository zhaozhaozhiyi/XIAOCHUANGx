## 2. 产品概述

Hermes Agent 是一个多入口、可扩展、可持续成长的 AI Agent 平台。它不是单纯的聊天机器人，也不是只服务于某一个 IDE 或某一个消息平台的轻量助手，而是一个可运行在终端、桌面端、Web 管理台和多种消息平台上的统一 Agent 系统。

平台的核心特征是：

- 同一个 Agent 内核可以在 CLI、TUI、Desktop、Web Dashboard、Telegram、Discord、Slack、WhatsApp 等多个入口复用
- 所有入口共享同一套配置、模型设置、技能、记忆、会话和自动化能力
- Agent 具备长期积累能力，不只是一次性回答，而是能够记忆、沉淀技能、跨会话复用经验
- 平台强调“能执行、能自动化、能扩展”，而不是停留在问答层

从工程结构和文档表达来看，Hermes Agent 的产品定位更接近“Agent 操作系统”或“多端统一 Agent 平台”。

## 3. 需求背景与要解决的问题

### 3.1 现有问题

传统 AI 助手或聊天式产品常见问题包括：

- 入口割裂：桌面、Web、终端、消息平台之间互不连通
- 能力短期化：每次对话都是一次性完成，缺少长期记忆与技能沉淀
- 自动化不足：擅长回答，但难以稳定执行复杂流程、定时任务或跨系统操作
- 扩展成本高：接入新模型、新工具、新平台需要单独开发
- 运维复杂：配置、日志、会话、定时任务、平台接入往往依赖命令行或手工编辑配置

### 3.2 Hermes Agent 的应对方式

Hermes Agent 通过统一 Agent 内核、多入口接入、技能系统、记忆系统、工具系统、定时调度、子代理委派和管理界面，试图将“AI 助手”升级为“可长期协作的执行型 Agent 平台”。

## 4. 目标用户

结合官网文档、桌面端文档和用户故事，可将目标用户大致归纳为以下几类：

### 4.1 个人高频 AI 用户

典型需求：

- 用终端或桌面端持续与 AI 协作
- 管理个人任务、文件、研究、写作和信息整理
- 通过语音、消息平台或自动化任务使用 Agent

### 4.2 开发者与技术团队

典型需求：

- 在本地或远程环境运行 Agent
- 让 Agent 具备代码、文件、命令、浏览器、MCP、API 集成等能力
- 通过技能、插件和多模型路由扩展系统能力
- 在 CLI、桌面端、Web 管理台之间共享工作流

### 4.3 自托管与自动化用户

典型需求：

- 在 VPS、Docker、SSH、Modal、Daytona 等环境部署 Agent
- 通过 cron、消息平台和 webhook 建立自动化流程
- 希望低成本长期运行，而不是必须依赖单一云服务前端

### 4.4 多渠道运营或组织协作用户

典型需求：

- 在多个消息渠道接入同一个 Agent
- 统一管理渠道配置、授权、会话、日志和投递
- 支持跨平台通知、客服、研究、内容生成和运维辅助

## 5. 核心使用场景

根据工程文档与 user stories，平台主要服务以下场景：

### 5.1 对话式协作

- 用户通过 CLI、TUI、桌面端或 Web 直接与 Agent 交互
- Agent 在对话中调用工具、编辑文件、浏览网页、执行任务
- 会话可跨端恢复

### 5.2 长期记忆与技能沉淀

- Agent 记住用户偏好、项目信息、工作习惯
- Agent 将复杂问题的解决方式沉淀成 skill
- 后续任务可自动加载相关技能和记忆

### 5.3 自动化执行

- 使用自然语言或 cron 表达式创建周期任务
- 自动投递结果到指定平台
- 定期执行研究、汇总、通知、备份、审计等任务

### 5.4 多平台消息代理

- 同一个 Agent 接入 Telegram、Discord、Slack、WhatsApp、Signal、Email 等
- 用户无需进入终端也能在常用沟通入口中使用 Agent

### 5.5 多工具与外部系统协作

- 通过 MCP、浏览器、API Server、插件、Provider Routing 与外部系统联动
- 接入代码库、数据库、浏览器、文档、第三方服务

### 5.6 桌面化与可视化管理

- 使用 Desktop 或 Web Dashboard 管理配置、密钥、模型、会话、日志、技能和定时任务
- 降低对命令行的依赖

## 6. 功能需求拆解

以下内容基于仓库现有功能文档整理。

### 6.1 多入口统一交互

需求目标：
让同一个 Agent 能在多个交互入口一致运行，并共享状态。

功能要点：

- CLI 交互
- TUI 交互
- Web Dashboard
- Desktop 原生客户端
- 多消息平台接入
- 会话跨入口恢复

价值：
降低用户切换成本，形成统一体验，而不是多个分裂产品。

### 6.2 Agent 能力系统

需求目标：
让 Agent 不只会回答，还能执行与扩展。

功能要点：

- Tools 与 Toolsets
- 技能系统 Skills
- 文件与上下文引用
- 代码执行
- 子代理委派
- 浏览器自动化
- 图像、语音、视觉能力

价值：
把问答产品升级为执行型、可编排型 Agent。

### 6.3 长期记忆与成长机制

需求目标：
让 Agent 随使用持续变强，而不是每次从零开始。

功能要点：

- Persistent Memory
- USER.md / MEMORY.md
- 会话检索与召回
- 技能自动创建与改进
- 用户建模
- 外部记忆提供者接入

价值：
构建平台差异化壁垒，提升长期复用率和用户粘性。

### 6.4 自动化与调度

需求目标：
让 Agent 可以无人值守地持续工作。

功能要点：

- Cron 任务创建与管理
- 定时投递到不同平台
- Hook 与事件扩展
- 批处理能力

价值：
把 Agent 从“对话工具”提升为“流程执行器”。

### 6.5 平台集成与扩展

需求目标：
支持多模型、多平台、多工具、多部署模式。

功能要点：

- 多模型提供商接入
- Provider Routing / Fallback
- MCP 接入
- API Server
- 插件系统
- Memory Provider / Context Engine 扩展

价值：
形成开放平台能力，降低对单一厂商和单一模型的依赖。

### 6.6 管理与运维

需求目标：
让复杂 Agent 系统具备可管理性和可观察性。

功能要点：

- 配置管理
- API Keys 管理
- Sessions 管理
- Logs 查看
- Analytics 分析
- Gateway 状态查看
- Profiles 管理

价值：
支持长期运行、多人使用和复杂部署，而不仅是开发演示。

## 7. 核心产品价值

### 7.1 统一的 Agent 运行底座

CLI、桌面端、Web 和消息平台不是多个孤立产品，而是同一个 Agent 的不同入口。共享配置、会话、技能和记忆，是平台最重要的结构性价值之一。

### 7.2 长期成长能力

仓库对“self-improving agent”反复强调，说明产品核心差异化不在“能不能调用工具”，而在“能不能越用越强”。

### 7.3 从对话到执行

通过工具、代码执行、子代理、MCP 和自动化调度，平台把“AI 聊天”扩展为“AI 执行与协作”。

### 7.4 高扩展性与自托管友好

面向开发者、研究者和高级用户，支持多平台部署和多种扩展模式，适合构建个人 Agent 环境，也适合团队内部能力平台。

## 8. 差异化分析

与普通聊天产品相比，Hermes Agent 的差异点主要在：

- 强调长期记忆与技能演进，而非单次回答
- 强调多入口统一，而不是单一界面
- 强调自托管、远程运行和多环境部署
- 强调消息平台、自动化和 cron 的结合
- 强调 MCP、插件、Provider Routing 等平台化能力
- 强调桌面端和 Web 管理台的完整运维管理能力

## 9. 非功能需求

结合工程设计，可以归纳出以下非功能需求：

### 9.1 可扩展性

- 支持插件
- 支持新模型提供商
- 支持新消息平台
- 支持新记忆后端
- 支持新工具与 skill

### 9.2 可部署性

- 支持本地安装
- 支持远程部署
- 支持 VPS / Docker / SSH / Modal / Daytona
- 支持桌面端与 Web 模式

### 9.3 可管理性

- 提供 Dashboard 和 Desktop 可视化入口
- 可查看日志、会话、状态与配置
- 可管理平台密钥和渠道接入

### 9.4 可恢复性

- 支持 checkpoint 与 rollback
- 支持会话恢复
- 支持长期运行与后台任务

### 9.5 安全性

- 提供命令审批机制
- 提供配对与授权控制
- 区分本地与远程访问模式
- 对 dashboard 暴露和远程连接给出安全约束

## 12. 结论

Hermes Agent 的本质是一个统一、多入口、可成长、可扩展的 Agent 平台。其核心价值不在单点聊天能力，而在以下组合能力：

- 同一个 Agent 在多入口共享状态
- 长期记忆与技能沉淀
- 工具调用、执行、委派与自动化
- 强扩展性与自托管能力
- 桌面端与 Web 的管理闭环

从文档现状看，工程中已经具备很强的产品说明和技术文档基础，但尚未形成传统意义上的统一中文需求文档。本文可作为该类中文需求分析文档的初版。
