# 写作 Skill 路由建议

## 目标

在 `skill-writing-base` 完成需求收敛后，选择一个最合适的主写作 Skill。

## 路由规则

| 场景 / 信号 | 优先 Skill |
|-------------|------------|
| 一般研究、分析、综合稿件 | `skill-writing-general` |
| 通知、请示、报告、函、意见、公文体例 | `skill-writing-official-doc` |
| 会议纪要、讨论记录整理 | `skill-writing-meeting-minutes` |
| 政策原文解读、法规影响分析 | `skill-wr-policy` |
| 专题研究、单一研究问题深挖 | `skill-wr-topic` |
| 行业研究 | `skill-wr-industry` |
| 宏观数据解读 | `skill-wr-macro` |
| 行业数据点评 | `skill-wr-sector-data` |

## 选择原则

1. 先看文稿用途和体裁，再看风格偏好。
2. 一次只选一个主写作 Skill。
3. 若用户明确指定体裁或模板，以用户指定为准。
4. 若用户只说“帮我写一篇”，默认退回 `skill-writing-general`。

## 交接要求

把以下内容带给后续主写作 Skill：

- 已确认的写作需求摘要
- 用户提供的材料路径
- 篇幅与风格偏好
- 必须保留内容
- 是否需要最终导出 DOCX
