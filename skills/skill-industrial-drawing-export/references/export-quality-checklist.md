# 导出质量检查清单

## 通用检查

导出前必须确认：

- 当前 `.scad` 已保存。
- 当前参数已写入 `drawing.parameters.json`。
- 导出目录为当前图纸目录下的 `exports/`。
- 旧导出不会被误认为当前参数的结果。

导出后必须确认：

- 文件存在且非空。
- 文件扩展名与格式一致。
- README 或参数 JSON 记录导出状态。
- 最终回复中的路径与实际路径一致。

## STL 检查

STL 应满足：

- 能被工作区 STL 预览器打开。
- 几何不是空模型。
- 主体比例与需求一致。
- 不是把源码文本或错误日志写成 `.stl`。

如使用 `exports/preview.stl`：

- 只能称为“预览 STL 草模”。
- 不能称为“正式 STL 导出”。
- 必须标注来源为 fallback 或平台预览生成。

## DXF 检查

DXF 应满足：

- R12 DXF 或工作区预览器可解析格式。
- 至少包含可见轮廓实体。
- 俯视投影不为空白页。
- 对孔、外轮廓或安装位置有基本表达。

如果使用参数轮廓兜底：

- 文件可以用于沟通轮廓，不承诺为加工级工程图。
- `drawing.parameters.json` 必须记录 `dxfMethod: "parameter_outline"`。
- 用户可见反馈要说明这是兜底导出。

## SCAD 检查

`.scad` 是主交付：

- 必须完整可打开。
- 不包含 Markdown 围栏。
- 顶部参数可读。
- 文件内保留模块化结构。
- 后续修改应优先编辑它，而不是编辑 STL / DXF。

## README 检查

`README.md` 应简短记录：

- 图纸对象。
- 关键尺寸。
- 可编辑参数。
- 已生成文件。
- 未执行或失败的导出原因。

## 回复检查

最终回复只需简短说明：

- 已生成哪些文件。
- 主文件路径。
- 预览 / 导出来源。
- 可以继续修改哪些参数。

禁止：

- 粘贴大段源码。
- 声称未生成的格式已完成。
- 混淆 fallback 与真实 OpenSCAD 导出。
