# P0 主线模块收口清单

| 属性 | 内容 |
|------|------|
| 文档版本 | v1.1 |
| 日期 | 2026-06-27 |
| 范围 | 当前六模块导航中的主验收线：**对话 / 写作 / PPT**；导航同时可见 **3D绘图 / 视频 / 推演** |
| 目标 | 在 Desktop Alpha / Beta 过渡期，先把主线模块的“真实链路可验收”收口清楚，并确认六模块入口不回退 |
| 不含 | 翻译、会议纪要、知识库、多人协作、云端沙箱完整闭环；3D / 视频 / 推演按各自子线单独验收 |

---

## 1. P0 目标

本轮 P0 只做三件事：

1. 当前产品定义彻底收口为六模块可见。
2. 对话 / 写作 / PPT 主线各自有一条可执行、可复测的真实链路验收路径。
3. 至少有一组最小回归测试，覆盖主线模块页面入口、主发送链路，以及 3D / 视频 / 推演入口可访问。

---

## 2. 当前模块定义

当前导航可见模块为：

- 对话
- 写作
- PPT
- 3D绘图
- 视频
- 推演

以下能力不属于当前现行模块：

- 翻译
- 会议纪要
- 知识库

验收要求：

- 侧边导航出现六模块，且不出现翻译、会议纪要、知识库。
- 路由保留 `/chat`、`/writing`、`/ppt`、`/3d`、`/video`、`/simulation` 主路径。
- Companion / contracts / module registry 不再把翻译视为现行模块。
- 3D / 视频 / 推演入口可打开共享聊天壳，但真实业务闭环按子线文档验收。

---

## 3. 真实链路验收

### 3.1 对话

必验链路：

1. 新建对话并发送消息。
2. 收到真实或 mock 流式回复。
3. 会话创建成功，URL 进入 `/chat/{id}`。
4. 工作区或项目绑定信息可正常保留。

最小通过标准：

- 能发送
- 能回流
- 能落会话
- 错误态不静默

### 3.2 写作

必验链路：

1. 进入 `/writing/new`。
2. 看到写作专属首页文案与 Skill 选择。
3. 发送首条消息后进入 `/writing/{id}`。
4. 写作会话沿用统一聊天主链路，不出现路由或状态异常。

Beta 继续验收项：

- 需求采集卡
- 大纲/正文生成
- `.md` 落盘
- `docx` 导出

### 3.3 PPT

必验链路：

1. 进入 `/ppt/new`。
2. 看到 PPT 专属首页文案与 Skill 选择。
3. 发送首条消息后进入 `/ppt/{id}`。
4. PPT 会话沿用统一聊天主链路，不出现路由或状态异常。

Beta 继续验收项：

- 需求采集卡
- 页纲生成
- `.html` / `.pptx` 交付物
- 预览与下载

---

## 4. 最小回归矩阵

P0 最小矩阵要求至少覆盖以下 5 项：

| ID | 模块 | 类型 | 说明 |
|----|------|------|------|
| R1 | 对话 | E2E | 新建对话并发送消息 |
| R2 | 对话 | E2E | 项目绑定与工作区展开 |
| R3 | 写作 | E2E | `/writing/new` 进入并发送首条消息 |
| R4 | PPT | E2E | `/ppt/new` 进入并发送首条消息 |
| R5 | 3D绘图 | E2E / smoke | `/3d/new` 可进入共享聊天壳 |
| R6 | 视频 | E2E / smoke | `/video/new` 可进入共享聊天壳 |
| R7 | 推演 | E2E / smoke | `/simulation/new` 可进入共享聊天壳 |
| R8 | 平台 | 静态检查 | 前端类型检查通过 |

推荐命令：

```bash
pnpm -C web exec tsc --noEmit
pnpm -C web test:e2e
```

---

## 5. 本轮完成定义

本轮完成后，应满足：

- 当前产品文档口径已经统一成六模块可见
- 对话 / 写作 / PPT 各自都有明确入口和最小可用链路
- 3D / 视频 / 推演入口保持可见且不误标为完整闭环
- 至少一条写作、一条 PPT 的自动化回归已存在

---

## 6. 后续优先级

P0 之后优先补：

1. 写作真实产物链路验收
2. PPT 真实产物链路验收
3. 导出能力与交付物卡片验收
4. 3D M1、视频 0.x、推演 Beta 的子线 smoke 与发布闸门
5. 历史计划文档里的旧翻译口径清理
